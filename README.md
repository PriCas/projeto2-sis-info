Informações do Projeto
Sistema desenvolvido em portugol


 
Plano de trabalho
 
Atividade Integrada — Desenvolvimento de Software em Portugol Tema: Saúde

Objetivo da Atividade
Desenvolver um sistema em Portugol Web Studio aplicando lógica de programação, estruturas condicionais, estruturas de repetição, funções e boas práticas de desenvolvimento de software.
Além do desenvolvimento do sistema, os grupos deverão produzir:
Documentação do software;
Pitch de apresentação;
Plano de testes executado e registrado em planilha.

Tema do Projeto
Saúde
Cada grupo deverá escolher um tema relacionado à área da saúde e desenvolver um sistema funcional.

Sugestões de Temas
Os grupos poderão escolher temas como:
Controle de IMC
Controle de vacinação
Agenda de consultas
Monitoramento de pressão arterial
Controle de medicamentos
Controle de alimentação/dieta
Sistema de triagem hospitalar
Controle de hidratação
Monitoramento de exercícios físicos
Controle de pacientes
Também será permitido escolher outros temas relacionados à saúde.

Desafio do Projeto
O grupo deverá identificar uma situação-problema da área da saúde e criar um sistema que ajude na resolução dessa necessidade utilizando Portugol.
O sistema deverá:
Solicitar dados do usuário;
Utilizar estruturas condicionais;
Utilizar estruturas de repetição;
Utilizar funções;
Exibir resultados organizados;
Possuir menus e navegação funcional;
Tratar entradas inválidas;
Ser comentado e organizado.

Requisitos Obrigatórios do Sistema
1. Entrada de Dados
O programa deverá solicitar informações ao usuário, como:
Nome
Idade
Peso
Altura
Pressão arterial
Horários
Opções de menu
Quantidades
Dados de pacientes

2. Estruturas Condicionais
Utilizar:
se/senao
escolha/caso
Exemplos:
Verificar se o paciente está saudável;
Identificar níveis de risco;
Validar idade;
Exibir alertas.

3. Estruturas de Repetição
Utilizar pelo menos uma:
enquanto
para
faca-enquanto
Exemplos:
Cadastro de vários pacientes;
Repetição do menu principal;
Novas consultas;
Novos cálculos.

4. Funções
O sistema deverá possuir funções para organizar o código.
Exemplos:
Função para calcular IMC;
Função para exibir menu;
Função para validar dados;
Função para cadastrar paciente.

5. Funcionalidades Obrigatórias
O sistema deverá conter:
Menu principal;
Mínimo de 3 funcionalidades;
Tratamento de erro para opções inválidas;
Exibição clara das informações;
Encerramento correto do sistema.

3. Pitch — Apresentação do Projeto
